# LB 模組需求清單（RQ0）

**來源**：需求說明書（RQ0）+ `_refs/07-標籤模組.md`（分析資料）
**模組代碼**：LB（標籤列印）
**程式代號**：LBSB01-標籤服務程式

## 模組概述

### 角色表

| 角色 | 說明 |
|------|------|
| 系統管理員 | 設定印表機、列印主機、標籤樣板 |
| 標籤列印管理人員 | 監控佇列、處理離線項目、查詢列印紀錄 |
| 外部系統（API 呼叫端） | 各業務模組（BC/CP/TL/BS）透過 API 傳入列印指令 |
| 系統（自動） | 佇列消化、故障移轉、版本更新 |

### 模組定位

標籤列印管理模組**僅負責標籤列印服務端**，各業務模組的列印呼叫由各自模組處理。

| 負責（LB 模組） | 不負責（由各業務模組處理） |
|---|---|
| 接收並驗證列印指令 | 決定何時呼叫列印 API |
| 佇列管理（線上/離線） | 組裝標籤變動資料（JSON） |
| 驅動印表機輸出標籤 | 選擇使用哪個標籤樣板 |
| 列印紀錄與稽核 Log | 列印結果的業務後續處理 |
| 印表機/主機配置管理 | 標籤內容的業務邏輯驗證 |
| 標籤樣板版面管理 | — |

### 作業流程

```
API 列印請求進入 → 驗證指令封包 → 進入線上佇列(FIFO) → 排程列印
                                                        ├ 成功 → 記錄 Log
                                                        └ 失敗 → 移至離線佇列 → 等待人工介入
```

### 跨模組介接

| 業務模組 | 代碼 | 呼叫情境 |
|---------|------|---------|
| 採血作業 | BC | 採血完成時列印採血標籤 |
| 成分管理 | CP | 製程/核血時列印成分標籤 |
| 檢驗作業 | TL | 檢體處理時列印檢驗標籤 |
| 供應作業 | BS | 供應出貨時列印供應標籤 |

### C — 佇列管理機制

| 編號 | 需求說明 | 操作角色 | 來源 |
|------|----------|----------|------|
| RQLB007 | 線上佇列（Online Queue）：所有接收到的列印項目優先進入此佇列，依序排隊等待輸出。Online Queue 為唯一驅動印表機列印的佇列。 | 系統自動 | 需求說明書 |
| RQLB008 | 離線/異常佇列（Offline Queue）：若線上佇列中的項目列印失敗（印表機未開機/卡紙/離線），系統需自動將該項目移至離線佇列，確保線上佇列其餘作業能繼續執行，不受單一錯誤卡單影響。 | 系統自動 | 需求說明書 |
| — | 三個 Queue 皆持久化於本地 SQLite（檔案型資料庫），確保斷電或程式異常重啟後資料不遺失。 | 系統自動 | 分析資料 |
| — | Queue 不實作在中央 DB 的理由：LBSB01 為獨立列印主機，斷網時 Local LAN 仍通，仍需提供列印服務；若 Queue 在中央 DB，斷網即無法取得待印資料。 | — | 分析資料 |
| — | DB Log Queue（第三層佇列）：任何指令進入、異動、完成、刪除時，同步寫入本地 DB Log Queue，由背景 Thread 非同步呼叫中央 API 寫入 DB。斷網時累積於本地，恢復網路後自動批次回寫。 | 系統自動 | 分析資料 |
| — | DB Log Queue 設計理由：(1) 明確記錄 Client 傳進來的指令 (2) 指令可被修改覆蓋，所有異動記錄在 DB 同一筆 Record (3) 斷電保護——一進來就先寫，不等列印完才記錄 (4) 斷網容錯——本地持久化，恢復後回寫 | — | 分析資料 |
