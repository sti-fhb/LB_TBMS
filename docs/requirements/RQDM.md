# 文件管理模組（DM）需求文件

> **模組代碼**：DM（Document Management，文件管理）
>
> **定位**：獨立於 TSBMS 主系統部署之文件管理服務，採主系統相同技術棧（FastAPI + React）開發，但自成一服務。透過 DP 模組提供之 API 認證取得 Token 後運作。
>
> **歷史**：原為獨立模組 → 2026-03-25 併入 ET（教育訓練） → 2026-04-24 因採自建路線、不走 Moodle Wiki/Book，重新獨立為 DM 模組。

---

## 1. 模組概述

### 1.1 角色

| 角色 | 職責範圍 |
|------|----------|
| 資訊人員（SOP_AUTHOR）| 上傳文件、撰稿、送審 |
| 文件審查者（SOP_REVIEWER）| 審核送審文件、核准或退件 |
| 使用者（SOP_READER）| 瀏覽、下載已發布文件、查閱線上操作手冊 |
| 系統自動 | 版本歷程維護、通知發送、稽核軌跡記錄 |

### 1.2 作業流程

```
文件撰稿與發布
──────────────
1. 資訊人員登入 DM（透過 DP API 認證取得 Token）
2. 於文件管理頁面選擇分類（SOP / 手冊 / 教材 / 表單）
3. 上傳 PDF/Word 或使用線上編輯器撰寫
4. 填寫版本號（v1.0 / v2.0）、變更摘要
5. 送交簽核 → 系統通知對應 SOP_REVIEWER
6. 審核者檢視、核准或退件（帶回意見）
7. 核准後系統保留舊版為歷程、發布新版、通知訂閱者

查閱
────
8. 使用者於主系統功能畫面觸發線上操作手冊 → 主系統 call DM API（帶 func_name）→ DM 回應對應手冊頁面
9. 使用者於 Moodle 教材內點 SOP 連結 → 轉跳 DM 文件頁瀏覽
10. 使用者於 DM 首頁 / 分類頁搜尋文件、瀏覽版本歷程
```

### 1.3 整合邊界

| 接點 | 方向 | 用途 |
|------|------|------|
| DP API（認證）| DM → DP | DM 以 TSBMS 帳密換取 Token（方案 B，同 Moodle）|
| DP API（User / Audit）| DM → DP | 查 User 資料、寫 Audit Log |
| 主系統功能畫面 → DM | 主系統 → DM | 開啟線上操作手冊（by func_name）|
| Moodle 教材連結 → DM | Moodle → DM | 教育訓練教材中連結至 SOP / 手冊查閱 |
| DM 通知 → Email / 主系統 | DM → 外部 | 文件發布 / 簽核通知 |

### 1.4 可配置參數

- 文件分類代碼（SOP / 手冊 / 教材 / 表單...）
- 簽核流程規則（單層 / 會簽）
- 版本號規則（語意 / 流水）
- 檔案格式限制、單檔大小上限
- 通知範本

## 8. 跨模組介接

| 呼叫方 | 目標 | 方式 | 用途 |
|--------|------|------|------|
| DM | DP（API）| HTTP + Token | 認證、查 User、寫 Audit Log |
| 主系統各模組 | DM（API）| HTTP + Token | 線上操作手冊查閱（by func_name）|
