## 1. 系統概述

LBSB01 標籤服務程式是三總捐供血系統（TSBMS）的標籤列印服務，部署於各站點的工作機上，提供 24 小時不間斷運作。

### 1.1 主要功能

- **線上列印**：接收 Client 端傳入的列印指令（Online Queue），即時列印標籤
- **離線重印**：將項目移至離線區（Offline Queue），供重印或延後處理
- **印表機管理**：新增、修改、刪除站點內的標籤印表機設定
- **測試列印**：在不影響線上排隊的情況下，使用測試頁驗證列印效果

### 1.2 三層 Queue 架構

| Queue | 用途 | Status |
|-------|------|--------|
| **Online Queue** | Client 端傳入的正式列印指令 | 0（待列印）|
| **Offline Queue** | 手動移入的離線/重印項目 | 2（離線區）|
| **DB Log** | 所有異動紀錄回寫中央 DB | 1（終態：印出後或刪除）|

### 1.3 列印 Log 狀態碼（Status）

| Status | 語意 | 說明 |
|--------|------|------|
| **0** | 待列印 | 項目在 Online Queue 中等候列印 |
| **1** | 終態 | 印出完成 或 被人工刪除（具體動作由 RES_ID 備註區分） |
| **2** | 離線區 | 項目在 Offline Queue 中等候重印 |

## 2. 啟動與認證

### 2.1 啟動程式

在站點工作機上執行：
```
python main.py
```

程式啟動後**自動以 APIDP001 進行認證**，不會顯示登入畫面。

### 2.2 認證機制

LBSB01 以「外部系統」身分透過 **APIDP001-外部系統資料接收介面** 自動認證：

| 項目 | 說明 |
|------|------|
| **API 端點** | `POST /api/ext/auth/token` |
| **認證 CODE** | `LB_PRINT` |
| **認證 PASSCODE** | `stark123` |
| **EA 對應元素** | `{1BEF51C7-CD73-44e6-8D3B-CD134B3D388D}` — 標籤印表機服務登入 |
| **TOKEN 有效期** | 由系統參數 `EXT_TOKEN_EXPIRE` 控管（預設 3600 秒） |

認證成功後，主畫面標題列會顯示：`LBSB01-標籤服務程式 [站點名] 【線上】`

認證失敗時標題列顯示：`LBSB01-標籤服務程式 [站點名] 【離線】`，程式以離線模式繼續執行。

### 2.3 本機設定檔（config.ini）

站點與認證設定存放於**執行檔同目錄**下的 `config.ini`（編譯為 .exe 後亦同）：

```ini
; LBSB01 標籤服務程式設定檔
; [site] 與 [api] 由管理者設定；[token] 由程式自動回寫

[site]
site_id = S01
site_name = 總院捐血中心

[api]
url = http://localhost:8000/api/ext/auth/token

[token]
; 以下由程式認證後自動回寫，不需手動編輯
value =
expires_in = 3600
```

| Section | Key | 說明 | 誰維護 |
|---------|-----|------|--------|
| `[site]` | `site_id` | 本站點代碼（**首次部署時須設定**） | 管理者 |
| `[site]` | `site_name` | 站點中文名稱 | 管理者 |
| `[api]` | `url` | APIDP001 認證端點 URL | 管理者 |
| `[token]` | `value` | 認證後的 TOKEN | 程式自動 |
| `[token]` | `expires_in` | TOKEN 有效秒數 | 程式自動 |

#### 首次部署設定步驟

1. 執行 `LBSB01.exe`（或 `python main.py`），程式會在執行檔同目錄自動建立 `config.ini`
2. 關閉程式
3. 以**記事本或任意文字編輯器**開啟 `config.ini`
4. 修改 `[site]` 區段：
   - `site_id`：填入本站點代碼（例如 `S01`、`S02`、`S03`），須與中央 DP 系統的站點代碼一致
   - `site_name`：填入站點中文名稱（例如 `總院捐血中心`）
5. 修改 `[api]` 區段：
   - `url`：填入中央 DP 認證端點的完整 URL（例如 `http://192.168.1.100:8000/api/ext/auth/token`）
   - 若中央服務使用 HTTPS，則填入 `https://...`
6. **不要修改** `[token]` 區段（由程式自動管理）
7. 存檔，重新啟動程式
8. 觀察標題列是否顯示 `【線上】`，確認認證成功

#### 各站點 config.ini 範例

**總院捐血中心（S01）**：
```ini
[site]
site_id = S01
site_name = 總院捐血中心

[api]
url = http://192.168.1.100:8000/api/ext/auth/token
```

**北院區（S02）**：
```ini
[site]
site_id = S02
site_name = 北院區

[api]
url = http://192.168.1.100:8000/api/ext/auth/token
```

> **注意**：各站點的 `[api] url` 通常指向同一台中央 DP 服務主機，僅 `[site]` 區段不同。

### 2.4 認證失敗（離線模式）

認證失敗時彈出警告訊息（含失敗原因），**程式不會結束**，改以**離線模式**繼續運行。

**線上 vs 離線差異**：

| 功能 | 線上模式 | 離線模式 |
|------|---------|---------|
| 本地 Queue 操作 | 可用 | 可用 |
| 中央 DB 存取（SRV） | 可用 | 不可用 |
| 印表機設定（CRUD） | 可用 | 僅讀取本地快取 |
| 標題列狀態 | `【線上】`（綠色） | `【離線】`（紅色） |

常見失敗原因：
- 中央 DP 服務未啟動或網路不通
- `config.ini` 的 `[api] url` 設定錯誤
- 認證 CODE / PASSCODE 不正確（需由 DP09-系統參數維護作業建立）

> **注意**：
> - LBSB01 24x7 常駐，認證 Session 在程式運行期間持續有效
> - TOKEN 過期後由程式自動重新換發（透過 APIDP001）
> - 認證憑證（CODE / PASSCODE）由系統管理者於 DP 系統參數中管理，不需操作人員介入

## 3. 主畫面操作

### 3.1 畫面佈局

主畫面由上而下分為五個區域：

| 區域 | 位置 | 說明 |
|------|------|------|
| **標題列** | 頂部 | 程式名稱、Auto 自動列印開關、系統時間 |
| **Online 區** | 上半（左+右）| 線上列印項目明細（左）+ Online Queue 清單與列印按鈕（右）|
| **紙張規格** | 中間 | 標籤類型、紙張尺寸、位移、明暗設定 |
| **Offline 區** | 下半（左+右）| 離線列印項目明細（左）+ Offline Queue 清單（右）|
| **訊息區** | 底部 | 操作紀錄與測試列印 |

### 3.2 選單功能

| 選單 | 子項 | 說明 |
|------|------|------|
| **設定** | 標籤印表機設定 | 開啟印表機設定頁面（Modal 視窗） |
| **標籤測試頁** | — | 開啟標籤測試頁（Modal 視窗） |

> Modal 視窗開啟期間，主畫面無法操作，須先關閉子視窗。

### 3.3 Online Queue（線上排隊列印）

**顯示內容**：從 Client 端經由 SRVLB001 傳入的待列印指令。

**操作**：

| 操作 | 方法 | 結果 |
|------|------|------|
| **查看明細** | 點選清單中某一筆 | 左側明細區顯示：時間序、條碼種類、張數、列印者、印表機編號、UUID |
| **列印** | 按「**列印線上指定項目**」（橘色大按鈕）| 送印，Status 0→1，RES_ID 記錄列印參數 |
| **移至離線** | 對清單項目**雙擊** | 該筆移到 Offline Queue，Status 0→2，RES_ID 備註 `-OffLine` |
| **刪除** | 按「**刪除單筆**」 | 該筆 Status 0→1，RES_ID 備註 `-Delete`（Soft Delete） |
| **Auto 自動列印** | 勾選「Auto自動依序列印」 | 系統自動依序處理 Online Queue 中的項目 |

**指定印表機**：在左側明細區下方的「指定印表機」下拉可切換要使用的印表機。

### 3.4 Offline Queue（離線等待重印）

**顯示內容**：從 Online 移入的項目、或由標籤測試頁以 Status=2 進件的測試項目。

**操作**：

| 操作 | 方法 | 結果 |
|------|------|------|
| **查看明細** | 點選清單中某一筆 | 左側離線明細區顯示詳情 |
| **移回線上** | 對清單項目**雙擊** | 該筆移回 Online Queue，RES_ID 備註 `-OnLine` |
| **重印** | 選取後送印 | Status 2→1，RES_ID 記錄列印參數 |
| **刪除** | 按「**刪除單筆**」 | Status 2→1，RES_ID 備註 `-Off_DEL` |
| **變更印表機** | 在明細區「變更待印印表機」下拉選擇 | 變更該筆離線項目要使用的印表機 |

> **注意**：Queue 異動時畫面自動刷新。

### 3.5 紙張輸出規格

| 欄位 | 說明 |
|------|------|
| **標籤** | 下拉選擇標籤類型（CP01~TL01，共 16 種） |
| **尺寸** | 自動帶入該標籤的預設尺寸 |
| **寬 / 高** | 可手動微調（單位 mm） |
| **左位移 / 上位移** | 列印位移校正（單位 dots） |
| **明暗** | 列印濃度（0=預設 12，最大 19；小型機建議 12~14，工業型建議 6~8） |
| **固定參數** | 勾選後切換標籤時不自動更改尺寸，方便固定某組設定 |

### 3.6 列印測試資料（底部）

在底部「列印測試資料」區域，可直接輸入文字並送印，用於快速驗證印表機連線。

**格式**：
- `\F##;` — 設定字型大小（如 `\F40;` 表示 40px）
- `\n` — 換行

**範例**：`\F40;測試標籤列印\n第二列測試`

### 3.7 訊息區

底部訊息區以時間戳記顯示最近的操作紀錄（列印結果、錯誤訊息等），最多保留 100 筆。

## 4. 印表機設定

從主畫面選單「設定 → 標籤印表機設定」開啟。

### 4.1 頁面說明

標題列顯示：`LBSB01-標籤印表機設定 [站點 Sxx-站點名]`

僅列出**登入站點**的印表機（由 SRVLB093 依 site_id 查詢）。

### 4.2 上方按鈕列

| 按鈕 | 說明 |
|------|------|
| **重新整理** | 重新從中央載入該站點的印表機清單 |
| **列印印表機識別貼紙** | 列印選取印表機的識別貼紙（待實作） |
| **新增** | 新增一台印表機（見 4.3） |
| **刪除** | 刪除選取的印表機（見 4.5） |

右上角提示：
- `PS: 明暗=0 代表預設 12，最大 19`
- `建議值: 小型機 12~14，工業型 6~8`

### 4.3 新增印表機

1. 按「**新增**」→ 清單尾端出現一筆空白列
2. **印表機編號**欄位自動解鎖（綠框），可自由填入
   - 編號須**唯一**，存檔時由 SRVLB091 檢查，重複則回傳錯誤
3. 填入其他欄位：
   - **名稱**：印表機說明名稱
   - **型號**：從下拉選擇 GoDEX 型號
   - **站點**：預設為登入站點
   - **Server IP**：預設為本機 IP（LBSB01 所在主機）
   - **Driver / 印表機 IP**：二擇一（互斥，見 4.6）
   - **左位移 / 上位移 / 明暗**：預設值 0 / 0 / 12
   - **說明**：備註
4. 按「**存 檔**」→ 呼叫 SRVLB091 新增

> **放棄新增**：未存檔時若點選其他列，空白列自動取消。

### 4.4 編輯印表機

1. 在清單中點選要修改的印表機
2. 下方編輯區自動帶入現有設定（**印表機編號**不可修改）
3. 修改欄位後按「**存 檔**」→ 呼叫 SRVLB094 更新

### 4.5 刪除印表機

1. 在清單中選取要刪除的印表機
2. 按「**刪除**」
3. 彈出確認訊息：

   > **刪除時會一併刪除 資訊設備有參考到的標籤對應資料!!**  
   > **請確認是否繼續？**

4. 按「是」確認後，系統依序執行：
   - **Step 1**：呼叫 SRVDP018 刪除 DP_COMPDEVICE_LABEL 中該站點 + 印表機的對應資料
   - **Step 2**：呼叫 SRVLB092 硬刪除 LB_PRINTER 該筆
5. 任一步驟失敗則中止，顯示錯誤訊息

### 4.6 Driver 與印表機 IP 互斥規則

Driver 和印表機 IP 是**互斥**欄位，同時只能填其中一個：

| 連線方式 | Driver 欄位 | 印表機 IP 欄位 |
|---------|------------|---------------|
| **USB 連線** | 填 `USB` | 留空白（自動鎖定） |
| **藍芽連線** | 填 印表機設定中的名稱 | 留空白（自動鎖定） |
| **IP 網路連線** | 留空白（自動鎖定） | 填 印表機 IP 位址 |

互斥同步：填入其中一欄後，另一欄會自動清空並鎖定為不可輸入。

### 4.7 印表機清單欄位

| 欄位 | 說明 |
|------|------|
| 印表機編號 | PK，唯一不可重複 |
| 名稱 | 印表機說明 |
| 站點 | 所屬站點代碼 |
| Server IP | LBSB01 服務主機 IP |
| 印表機 IP | 網路印表機位址（與 Driver 互斥） |
| Driver | 印表機連線驅動（USB / 藍芽名稱）（與印表機 IP 互斥） |
| 左位移 / 上位移 | 列印偏移校正（dots） |
| 明暗 | 列印濃度 |
| 型號 | GoDEX 印表機型號 |
| 啟用 | Y / N |
| 說明 | 備註 |

## 5. 標籤測試頁

從主畫面選單「標籤測試頁」開啟。

### 5.1 用途

在**不影響 Online Queue 正式排隊**的前提下，驗證標籤列印效果。

### 5.2 運作原理

測試頁走與 Client 端**相同的 SRVLB001-標籤列印通用 API**，唯一差別：

| 來源 | Status | 進入 Queue |
|------|--------|-----------|
| Client 端正式列印 | 0（待列印） | Online Queue |
| **標籤測試頁** | **2（離線區）** | **Offline Queue** |

### 5.3 操作步驟

1. 開啟標籤測試頁（Modal 視窗，主畫面暫不可操作）
2. 選擇標籤類型、設定列印參數
3. 填入測試資料
4. 按「列印」→ 經由 SRVLB001 以 Status=2 進件，直接進入 Offline Queue
5. 回主畫面可在 Offline Queue 看到該筆測試項目
6. 關閉測試頁即可回到主畫面操作

## 7. 常見問題排除

### 7.1 登入後看不到主畫面

- 可能登入對話框在背景，按 `Alt+Tab` 切換
- 確認密碼正確後再試

### 7.2 列印失敗：「DLL 錯誤」

- 確認 GoDEX EZPL DLL 已正確安裝於 LBSB01 所在工作機
- 確認 DLL 路徑在 Python 執行環境可找到

### 7.3 列印失敗：「連線失敗」

- **USB 連線**：確認印表機已開機、USB 線連接正常、Driver 欄填 `USB`
- **IP 連線**：確認印表機 IP 位址正確、網路可達（ping 測試）、印表機 IP 欄有填值
- **藍芽連線**：確認藍芽已配對、Driver 欄填寫正確的藍芽裝置名稱

### 7.4 印表機設定看不到印表機

- 印表機設定頁面僅列出**登入站點**的印表機
- 確認登入時選擇了正確的站點
- 按「重新整理」重新載入

### 7.5 新增印表機時編號重複

- SRVLB091 會檢查 LB_PRINTER 中編號唯一性
- 請使用不同的印表機編號重試

### 7.6 明暗值建議

| 印表機類型 | 建議明暗值 |
|-----------|-----------|
| 桌上型（G500, G530, RT700i） | 12 ~ 14 |
| 工業型（EZ2250i, EZ2350i, EZ6250i） | 6 ~ 8 |
| 攜帶型（DT2x, MX30） | 12 ~ 13 |
| 明暗值 = 0 | 系統使用預設值 12 |

### 7.7 詳細錯誤紀錄

列印失敗時詳細錯誤會記錄在程式目錄下的 `app.log`，可提供給系統管理者排查。

## 附錄 B：RES_ID 寫入格式

列印 Log（LB_PRINT_LOG）的 RES_ID 欄位記錄每次動作的完整資訊：

**格式**：`[VERSION] + ['F'?固定參數] + 'W'[寬] + 'H'[長] + 'L'[左位移] + 'T'[上位移] + 'D'[明暗值] + [備註]`

VERSION 為程式版本號（目前 `v1.1r1`，顯示於主畫面標題列）。

**備註代碼**：

| 代碼 | 意義 | Status 變更 |
|------|------|-----------|
| `-OnLine` | 移至 Online 區 | 保持 |
| `-OffLine` | 移至離線區 | → 2 |
| `-Delete` | Online 人工刪除 | → 1 |
| `-Off_DEL` | Offline 刪除 | → 1 |

**範例**：
- 已印出：`v1.1r1W80H35L40T0D8`
- 移至離線：`v1.1r1-OffLine`
