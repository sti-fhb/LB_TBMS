# ISLB003 — CP01 標籤移除 DATA_4 / 6 / 8 / 11

**建立日期**: 2026-04-17
**優先順序**: MEDIUM
**類型**: 規格變更
**狀態**: 待實作

---

## 1. 背景

CP01（血品小標籤，舊 L00）原本會在 info_line / extra_line 列印以下 4 項資料，經確認**不需要列印**：

| 欄位 | SPEC 用途 | 程式行為 |
|------|-----------|---------|
| `data_4` | 袋序（例 `1`） | 在 `bar_l00.py` 被當作 `"1"=[Special]` 旗標，組入 info_line |
| `data_6` | 處理方式（例 `分離`） | 組入 info_line |
| `data_8` | 採集方式（例 `MCS`） | 組入 info_line |
| `data_11` | 製備批號（例 `0607A6999`） | 組入 extra_line |

## 2. 設計決策

| 項目 | 新設計 |
|------|--------|
| CP01 列印內容 | 移除 `data_4 / 6 / 8 / 11` 的列印邏輯（info_line 與 extra_line） |
| CP02（L00A）是否受影響 | **不受影響**。`bar_l00.py` 同一函式服務 CP01/CP02，修改**僅限 non-L00A 分支**（`else:` 區塊） |
| SRVLB001 API 介面 | Client 端仍可傳這些欄位（向下相容），但 LBSB01 **不印出** |
| 欄位規格 | `SRVLB001-標籤列印整合指南.md` 的 CP01 欄位表**移除這 4 列** |
| 測試頁 sample data | `sample_data.py` 的 CP01 測試資料一併清空這些欄位 |
| 其他欄位 | 不動（data_1, 3, 5, 7, 10, 12, 19 保留原樣） |

## 3. 實作範圍

### 3.1 `Source/Python/LBSB01/bar_l00.py`

**目標分支**：`print_l00()` 函式中 **non-L00A 分支**（`else:` 區塊，約第 114-134 行）。

**變更前（第 114-134 行）**：

```python
else:
    parts: list[str] = []
    if data.data_8:
        parts.append(data.data_8)
    if data.data_4 == "1":
        parts.append("[Special]")
    if data.data_6:
        parts.append(data.data_6)

    info_line = ",".join(parts)
    printer.text_out(bx, py + 48, 48, FONT_0, info_line)

    # D10 + D11
    extra_parts: list[str] = []
    if data.data_10:
        extra_parts.append(data.data_10)
    if data.data_11:
        extra_parts.append(data.data_11)
    if extra_parts:
        extra_line = ",".join(extra_parts)
        printer.text_out(bx, py + 96, 48, FONT_0, extra_line)
```

**變更後**：

```python
else:
    # CP01: 不再列印 data_4/6/8 組成的 info_line（ISLB003）
    # 若後續 CP01 需要其他欄位組 info_line，另行加回

    # extra_line 只保留 data_10（data_11 移除，ISLB003）
    if data.data_10:
        printer.text_out(bx, py + 96, 48, FONT_0, data.data_10)
```

**注意**：
- `info_line` 整行移除（因 data_4/6/8 都是其組成來源，全部拿掉後整行為空）
- 若要保留版面留白（避免後續列印項目上移），可保留 `printer.text_out(bx, py + 48, 48, FONT_0, "")`；但 GoDEX 列印空字串通常等同跳過，建議直接移除 `text_out` 呼叫
- **L00A 分支不動**（`if is_type_a:` 的 `data_4`/`data_6` 呼叫保持原樣，服務 CP02）
