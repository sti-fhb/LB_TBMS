# ISLB002 — CP11 移除 DATA_8/13/14/15 + CP19 移除 DATA_13

**建立日期**: 2026-04-17
**優先順序**: MEDIUM
**類型**: 規格變更
**狀態**: 待實作

---

## 1. 背景

經樣本標籤確認，以下資料**不需要列印**：

### CP11（血品核對標籤-合格）

| 欄位 | 原本用途 | 說明 |
|------|---------|------|
| `data_8` | 抗原名稱（Mia/C/c/E/e/M/Jka/Jkb 等，每 7 碼一組） | 抗體篩檢區，確認不印 |
| `data_13` | 機構地址（含 Logo 圖片） | 發行單位區，確認不印 |
| `data_14` | 生物危害標誌（`"1"`=顯示） | 生物危害符號，確認不印 |
| `data_15` | 自體捐血（`"1"`=顯示） | 自體捐血直書文字，確認不印 |

### CP19（血品核對標籤-不適輸用）

| 欄位 | 原本用途 | 說明 |
|------|---------|------|
| `data_13` | 機構地址（含 Logo 圖片） | 發行單位區，確認不印 |

## 2. 設計決策

| 項目 | 新設計 |
|------|--------|
| CP11 列印內容 | 移除 `data_8 / 13 / 14 / 15` 的列印邏輯 |
| CP19 列印內容 | 移除 `data_13` 的列印邏輯 |
| SRVLB001 API 介面 | Client 端仍可傳這些欄位（向下相容），但 LBSB01 **不印出** |
| 欄位規格 | `SRVLB001-標籤列印整合指南.md` 的 CP11 / CP19 欄位表**移除對應列** |
| 測試頁 sample data | `sample_data.py` 的 CP11 / CP19 測試資料**一併清空**這些欄位（set to `""`），避免誤導 |
| 圖檔資產 | `images/biomedical-50.JPG`、`images/Logo-18.jpg` 保留（未來可能還會用到） |
| 其他欄位 | 不動（保留原樣） |
