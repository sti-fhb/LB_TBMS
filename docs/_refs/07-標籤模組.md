# LB-標籤模組 範圍描述

## 模組定位

標籤列印管理模組**僅負責標籤列印服務端**，各業務模組的列印呼叫由各自模組處理。

| 負責（LB 模組） | 不負責（由各業務模組處理） |
|---|---|
| 接收並驗證列印指令 | 決定何時呼叫列印 API |
| 佇列管理（線上/離線） | 組裝標籤變動資料（JSON） |
| 驅動印表機輸出標籤 | 選擇使用哪個標籤樣板 |
| 列印紀錄與稽核 Log | 列印結果的業務後續處理 |
| 印表機/主機配置管理 | 標籤內容的業務邏輯驗證 |
| 標籤樣板版面管理 | — |

## 主要流程

```
API 列印請求進入 → 驗證指令封包 → 進入線上佇列(FIFO) → 排程列印
                                                         ├ 成功 → 記錄 Log → 回傳 SUCCESS
                                                         └ 失敗 → 移至離線佇列 → 等待人工介入
```

## 跨模組介面

### API 介面（供各模組呼叫）

```
POST /api/print/label
```

| 參數 | 型態 | 說明 |
|------|------|------|
| targetHost | string | 目標列印主機 IP |
| printerNo | string | 印表機序列號（#1~#6） |
| templateCode | string | 標籤樣板代碼 |
| data | JSON | 標籤變動資料 |
| callerModule | string | 來源模組代碼（BC/CP/LI/DR） |

## 系統架構（標籤印表機）

> 來源：EA Model — 「標籤印表機系統架構圖」(Element ID: 1238646847)

### 架構概述

採用 **Printer Server（標籤伺服主機）** 架構，每台 Printer Server 為獨立運作的單機，透過 **FastAPI HTTP Server** 接收列印指令，再轉發至實體標籤印表機輸出。

### 架構圖（文字版）

```
┌─────────────────────────────────────────────────────────────┐
│  Client（瀏覽器端）                                          │
│  Client #1, #2~#n                                           │
│  ─ 登入後依「站點-標籤-印表機對應表」決定標籤印到哪台印表機      │
│  ─ BMS 功能要印標籤時，依 Printer Server IP call FastAPI      │
└────────────┬────────────────────────────────────────────────┘
             │ HTTP POST（call FastAPI）
             │
        ┌────┴────┐
        │  內網    │
        └────┬────┘
             │
┌────────────┴────────────────────────────────────────────────┐
│  Printer Server PC #1, #2~#n（獨立主機運作）                  │
│  ─ FastAPI 輕量 HTTP Server                                  │
│  ─ 接收列印指令 → 轉發至標籤印表機                             │
│  ─ 回寫 LOG 至內網 DB（BMS Table）                            │
│  ─ Pooling Print Pool TABLE 取得待印工作                      │
└────────────┬────────────────────────────────────────────────┘
             │
    ┌────────┼────────┐
    │        │        │
  USB      藍芽    TCP/IP
  固接    OS層連線   固定IP
    │        │        │
  Printer  Printer  Printer
```

### 運作機制

1. **Client 端（瀏覽器）**：使用者登入 BMS 後，系統依「**站點-標籤-印表機對應表**」得知該類標籤應印到哪台印表機及對應的 Printer Server IP
2. **發送列印指令**：BMS 功能需要印標籤時，Client 透過 HTTP POST 呼叫目標 Printer Server 的 FastAPI
3. **Printer Server 處理**：
   - 每台 Printer Server 為獨立單機運作
   - 透過 FastAPI HTTP Server 接收列印指令
   - 以 Pooling 方式輪詢 Print Pool TABLE 取得待印工作
   - 將指令轉發至連接的標籤印表機
4. **印表機連線方式**（三種）：
   - **USB** — 固接
   - **藍芽** — 透過 OS 層連線（Windows Printer Name）
   - **TCP/IP** — 固定 IP 直連
5. **LOG 回寫**：列印完成後，Printer Server 回寫 LOG 至內網 DB（BMS Table）

### 印表機定址方式

```
[Printer Server IP] + [#{Windows Printer Name} / 'USB']
[Printer Server IP] + {Printer IP}    // 若有填則優先採用
```

### 關鍵設計

- **去中心化**：每台 Printer Server 獨立運作，不依賴單一中央服務
- **彈性對應**：透過「站點-標籤-印表機對應表」設定，不同站點/標籤類型可指向不同印表機
- **支援多種連線**：USB、藍芽、TCP/IP 三種方式，適應不同場域硬體環境

## 硬體環境

- 標籤印表機：Godex 系列（熱轉印）
- 連線方式：USB、藍芽、TCP/IP
- 每主機最多 6 台印表機
- 標籤紙規格：45x15mm / 80x35mm / 80x75mm / 200x100mm / 90x15mm / 40x35mm

## 設計重點

1. **佇列隔離**：線上佇列 FIFO 排序，失敗項自動移至離線佇列，不阻塞後續列印
2. **服務端專責**：各業務模組只需呼叫 API，列印邏輯集中在 LB 模組
3. **完整稽核**：所有列印操作記錄 Log，可追溯任一血品標籤的完整列印歷程
4. **離線備援**：BMS 系統失效時，可手動輸入血品資料印出合格標籤
5. **自動版本更新**：閒置 10 分鐘自動更新，支援人工回退
